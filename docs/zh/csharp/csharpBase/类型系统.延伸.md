# 类型系统延伸

## .NET 的主要内存区域

.NET 主要有以下几个内存区域：

- 栈：存储值类型局部变量、方法参数、返回值为值类型的方法的返回值、引用类型的引用等。
- 托管堆：存储所有引用类型的对象实例。堆又可细分为：
    - GC堆：常规对象分配的区域。
    - 大对象堆：大型对象（大于85KB）的分配的区域。
    - 驻留池：字符串字面量的缓存区，也是托管堆的一部分，但由GC特殊管理。

> [!TIP] 提示
> **驻留池**是一个哈希表，它在进程生命周期内将会一直存在，存储着所有被驻留的字符串。当你使用相同的字符串字面量时，CLR会返回驻留池中的同一个引用。

## 关于字符串 (string) 的特殊存储

在有了上面相关知识点的基础以后，就可以更加深入的研究字符串的存储了。

### 字符串的赋值

首先是引用类型标准的赋值时为引用传递：

::: code-group

```cs [C#]
string s1 = "hello";
string s2 = s1;

```

:::

但是在给字符串重新赋值时，和其它引用类型的行为就有所不同了：

::: code-group

```cs [C#]
string s1 = "hello";
string s2 = s1;
s2 = "world"; // 新建一个字符串对象，并将引用赋值为s2
Console.WriteLine(Object.ReferenceEquals(s1, s2)); // false
```

:::

这时候, `s1` 和 `s2` 的引用已经完全不同了，所以在进行引用比较时，结果为 `false`。

## 字符串的存储

字符串的存储和其它引用类型并没有太大的区别，依旧是堆中存对象，使用引用指向堆中的对象。但是字符串有一片额外的被称为 `驻留池` 的缓存区，用来存储相同的字符串字面量。

当你声明一个字符串字面量时，CLR会先检查是否有相同的字面量存在于驻留池中，如果有，则直接返回引用；如果没有，则会在托管堆中创建一个新的字符串对象，并将引用返回。

**前提是这个字符串不是动态构造出来的，如果是，那么必须手动调用 `string.Intern` 方法让字符串进入驻留池**。

::: code-group

```cs [C#]
// 非动态构建
string s1 = "hello"; // 将字面量 "hello" 存入托管堆的驻留池中
string s2 = "hello"; // 直接返回托管堆中驻留池中的 "hello" 引用
Console.WriteLine(Object.ReferenceEquals(s1, s2)); // true，因为引用相同

// 动态构建
string s3 = "world";
string s4 = new string(new char[] { 'w', 'o', 'r', 'l', 'd' });
Console.WriteLine(Object.ReferenceEquals(s3, s4)); // false，因为是动态构建的
s4 = string.Intern(s4); // 手动调用 Intern 方法将 s4 字符串进入驻留池，然后将方法返回的引用赋值给 s4
Console.WriteLine(Object.ReferenceEquals(s3, s4)); // true，因为引用相同
```

:::
